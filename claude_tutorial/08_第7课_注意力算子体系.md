## 第一部分：基础设施

## 1. AttnWeightTemplate：无权重的算子容器

**核心文件**：`lightx2v/common/ops/attn/template.py`

第4课讲了 `WeightModule` 树中的叶子节点（MMWeight、TensorWeight），它们持有实际的权重张量。注意力算子是一个特殊的叶子——**它没有可学习的权重**，但需要嵌入 WeightModule 树中（因为树的遍历机制需要统一处理 `load`/`state_dict`/`to_cpu`/`to_cuda`）。

```python
# common/ops/attn/template.py
class AttnWeightTemplate(metaclass=ABCMeta):
    def __init__(self, weight_name):
        self.weight_name = weight_name
        self.config = {}

    def load(self, weight_dict):
        pass                          # 没有权重要加载

    @abstractmethod
    def apply(self, input_tensor):    # 唯一需要实现的方法
        pass

    def to_cpu(self, non_blocking=False):
        pass                          # 没有权重要搬运

    def to_cuda(self, non_blocking=False):
        pass

    def state_dict(self, destination=None):
        if destination is None:
            destination = {}
        return destination            # 导出空字典

    def load_state_dict(self, destination, block_index, adapter_block_inde=None):
        return {}                     # Offload 时无需传输
```

**设计要点**：

1. **空操作**：所有涉及权重的方法（`load`、`state_dict`、`to_cpu`、`to_cuda`）都是空操作。这意味着注意力算子在 Offload 过程中**不产生任何传输开销**——它是 Block 中唯一不需要 CPU↔GPU 搬运的组件。

2. **统一接口**：所有注意力实现必须实现 `apply()`，这保证了推理代码可以写成 `phase.self_attn_1.apply(q=q, k=k, v=v, ...)`，不关心底层是 Flash Attention 还是 Sage Attention。

3. **注册入口**：在 `WanSelfAttention.__init__()` 中通过 `ATTN_WEIGHT_REGISTER[config["self_attn_1_type"]]()` 创建具体实例并用 `add_module` 挂到 WeightModule 树上。

## 3. 配置路由：从 JSON 到算子实例

**关键路径**：`config.json` → `WanSelfAttention.__init__()` → `ATTN_WEIGHT_REGISTER[key]()`

Wan 模型的注意力类型由三个配置项决定：

| 配置项 | 用途 | 默认值 |
|--------|------|--------|
| `self_attn_1_type` | Self Attention（最大的计算瓶颈） | `flash_attn2` |
| `cross_attn_1_type` | Cross Attention（文本条件） | `flash_attn2` |
| `cross_attn_2_type` | Cross Attention（图像条件，仅 I2V） | `flash_attn2` |

**路由代码**（`wan/weights/transformer_weights.py:340-426`）：

```python
class WanSelfAttention(WeightModule):
    def __init__(self, ...):
        # 1. 通过 config key 取出注意力类
        attention_weights_cls = ATTN_WEIGHT_REGISTER[self.config["self_attn_1_type"]]

        # 2. 对特定算子做额外配置（类属性注入）
        if self.config["self_attn_1_type"] == "sla_attn":
            sla_config = self.config.get("sla_attn_setting", {})
            if "sparsity_ratio" in sla_config:
                attention_weights_cls.sparsity_ratio = sla_config["sparsity_ratio"]
            if "operator" in sla_config:
                attention_weights_cls.operator = sla_config["operator"]
        # ... 对 sparge_attn, spas_sage_attn2, nbhd_attn 等类似处理 ...

        # 3. 实例化并注册到 WeightModule 树
        self.add_module("self_attn_1", attention_weights_cls())
```

**为什么用类属性注入而非构造参数？**

注意 `attention_weights_cls.sparsity_ratio = ...` 设置的是**类属性**而非实例属性。这是因为 `AttnWeightTemplate.__init__` 的签名是固定的 `(self, weight_name)`，各实现类的 `__init__` 不接受额外参数。通过类属性注入，可以在不修改基类接口的情况下传递配置。

**配置文件示例**：

```json
// configs/attentions/wan_i2v_sla_480p.json
{
    "self_attn_1_type": "sla_attn",        // ← self-attn 用 SLA 稀疏注意力
    "sla_attn_setting": {
        "sparsity_ratio": 0.8,             // ← 80% 稀疏率（只计算 20% 的块）
        "operator": "triton"               // ← 用 Triton 内核做稀疏计算
    },
    "cross_attn_1_type": "flash_attn3",    // ← cross-attn 仍用 dense（文本序列短）
    "cross_attn_2_type": "flash_attn3"
}
```

**为什么 Cross Attention 通常用 Dense？**

Self Attention 的序列长度 = 视频 token 数（如 Wan 480P 81帧 ≈ 32760），是 O(L²) 计算的主要来源。Cross Attention 的 KV 来自文本编码器（最多 512 token），序列长度远小于视频，稀疏化收益不大。

## 4. Flash Attention 系列

**核心文件**：`lightx2v/common/ops/attn/flash_attn.py`

### 4.1 FlashAttn2Weight

```python
@ATTN_WEIGHT_REGISTER("flash_attn2")
class FlashAttn2Weight(AttnWeightTemplate):
    def apply(self, q, k, v,
              cu_seqlens_q=None, cu_seqlens_kv=None,
              max_seqlen_q=None, max_seqlen_kv=None, **kwargs):
        # 输入: q/k/v shape = [L, H, D]（varlen 格式）或 [B, L, H, D]
        if len(q.shape) == 3:
            bs = 1
        elif len(q.shape) == 4:
            bs = q.shape[0]
            q = q.reshape(-1, q.shape[-2], q.shape[-1])  # → [B*L, H, D]
            k = k.reshape(-1, k.shape[-2], k.shape[-1])
            v = v.reshape(-1, v.shape[-2], v.shape[-1])

        x = flash_attn_varlen_func(     # ← Flash Attention 2 的核心 API
            q, k, v,
            cu_seqlens_q, cu_seqlens_kv, # 累积序列长度（支持变长序列）
            max_seqlen_q, max_seqlen_kv,
        ).reshape(bs * max_seqlen_q, -1) # 输出: [L, H*D]
        return x
```

**关键点**：
- 使用 `flash_attn_varlen_func`（支持变长序列打包，无 padding 浪费）
- 输入 layout：`[L, H, D]`（每个 token 的每个 head 的特征）
- 输出 layout：`[L, H*D]`（heads 维度已 flatten）
- `cu_seqlens`：累积序列长度数组，如 `[0, 32760]` 表示一个长度为 32760 的序列

### 4.2 FlashAttn3Weight / FlashAttn4Weight

```python
@ATTN_WEIGHT_REGISTER("flash_attn3")
class FlashAttn3Weight(AttnWeightTemplate):
    # 结构与 FA2 完全一致，只是调用 flash_attn_varlen_func_v3
    ...

@ATTN_WEIGHT_REGISTER("flash_attn4")
class FlashAttn4Weight(AttnWeightTemplate):
    def apply(self, q, k, v, ...):
        # FA4 不支持 varlen，需要 [B, L, H, D] 格式
        if len(q.shape) == 3:
            q, k, v = q.unsqueeze(0), k.unsqueeze(0), v.unsqueeze(0)
        assert bs == 1, "flash_attn4 doesn't support varlen, batchsize must be 1"
        x, _ = flash_attn_func_v4(q, k, v)  # ← FA4 的 cute API

```

**FA2/FA3/FA4 的差异**：

| 特性 | FA2 | FA3 | FA4 |
|------|-----|-----|-----|
| API | `flash_attn_varlen_func` | `flash_attn_varlen_func_v3` | `flash_attn_func_v4` |
| 变长序列 | 支持（varlen） | 支持（varlen） | 不支持 |
| GPU 架构 | SM80+ (A100) | SM90 (H100) | SM90+ |
| 稀疏 mask | 不支持 | 不支持 | 支持（block_sparse） |

## 5. Sage Attention 系列

**核心文件**：`lightx2v/common/ops/attn/sage_attn.py`

### 5.1 SageAttn2Weight

```python
@ATTN_WEIGHT_REGISTER("sage_attn2")
class SageAttn2Weight(AttnWeightTemplate):
    def apply(self, q, k, v, ...):
        q, k, v = q.contiguous(), k.contiguous(), v.contiguous()
        # 输入: [L, H, D] → 需要扩展为 [B, L, H, D]
        if len(q.shape) == 3:
            q, k, v = q.unsqueeze(0), k.unsqueeze(0), v.unsqueeze(0)
        x = sageattn(q, k, v, tensor_layout="NHD")  # ← SageAttention API
        return x.view(bs * max_seqlen_q, -1)
```

**SageAttention 的核心思想**：用 INT8 量化 Q/K 矩阵的点积计算，用 FP16 计算 softmax 和 V 的加权求和。在保持精度的同时减少 FLOPS。

- **SageAttn2**：适配 SM80/SM89/SM120 架构（自动选择最优 kernel）
- **SageAttn3**：专门为 Blackwell (SM120) 架构优化，layout 是 `[B, H, L, D]`（注意 transpose）

## 6. Torch SDPA（兜底实现）

**核心文件**：`lightx2v/common/ops/attn/torch_sdpa.py`

```python
@ATTN_WEIGHT_REGISTER("torch_sdpa")
class TorchSDPAWeight(AttnWeightTemplate):
    def apply(self, q, k, v, drop_rate=0, attn_mask=None, causal=False, ...):
        # [L, H, D] → [1, L, H, D] → [1, H, L, D]（PyTorch SDPA 要求 BHLP layout）
        if q.ndim == 3:
            q, k, v = q.unsqueeze(0), k.unsqueeze(0), v.unsqueeze(0)
        q = q.transpose(1, 2)   # [B, L, H, D] → [B, H, L, D]
        k = k.transpose(1, 2)
        v = v.transpose(1, 2)
        x = F.scaled_dot_product_attention(q, k, v,
            attn_mask=attn_mask, dropout_p=drop_rate, is_causal=causal)
        x = x.transpose(1, 2)   # [B, H, L, D] → [B, L, H, D]
        return x.reshape(b, s, -1).squeeze(0)  # → [L, H*D]
```

这是纯 PyTorch 实现，**不依赖任何第三方库**。PyTorch 会在内部根据硬件自动选择 FlashAttention / Memory-Efficient Attention / Math 后端。适合调试和兼容性场景。

## 7. Dense 算子的输入/输出 Layout 对比

所有 Dense 注意力算子的 `apply()` 都接受 `[L, H, D]` 或 `[B, L, H, D]` 输入，返回 `[L, H*D]` 输出。但内部的 kernel 要求不同的 layout：

```
外部统一接口:   [L, H, D]  ──────────→  [L, H*D]
                   │                         ↑
           ┌──────┴──────────────┐          │
           │  各 kernel 内部 layout  │          │
           │                        │          │
  FA2/FA3: │  [L, H, D] (varlen)   │ ─reshape→│
  FA4:     │  [B, L, H, D]         │ ─reshape→│
  Sage2:   │  [B, L, H, D] (NHD)   │ ─view──→│
  Sage3:   │  [B, H, L, D]         │ ─reshape→│
  SDPA:    │  [B, H, L, D]         │ ─reshape→│
           └────────────────────────┘
```

**这个统一接口的价值**：推理代码（`infer_self_attn`、`infer_cross_attn`）不需要关心 layout 差异——它只需要把 Q/K/V 以 `[L, H, D]` 格式传入，拿到 `[L, H*D]` 输出即可。所有 layout 转换都封装在各算子的 `apply()` 内部。

## 第三部分：Sparse 注意力实现

视频生成的 Self Attention 是 O(L²) 的计算瓶颈。以 Wan 480P 81帧为例，L ≈ 32760，Attention Map 大小 = 32760² ≈ 10 亿个元素。稀疏注意力的核心思想是**只计算 Attention Map 中最重要的部分**，跳过不重要的区域。

稀疏注意力有两个独立的维度：
1. **Mask 生成策略**：决定"哪些 Q-K 对需要计算"（稀疏 pattern）
2. **计算后端**：在给定 mask 下，用什么 kernel 做实际的 Attention 计算

框架中提供了两种组织方式：

```
方式 A: 单体实现（Mask 与计算绑定）
  sla_attn, sparge_attn, spas_sage_attn2, ...
  ↳ 一个类内部同时做 mask 生成 + 稀疏计算

方式 B: 解耦实现（general_sparse_attn 框架）
  mask_generator × operator 自由组合
  ↳ 配置驱动，不改代码即可换 mask 策略或计算后端
```

## 8. 稀疏注意力的核心概念：Block Sparse

几乎所有稀疏注意力实现都基于**块稀疏**（Block Sparse）：把 Attention Map 划分为固定大小的块（如 128×128），以块为单位决定计算/跳过。

```
        K 序列（32760 tokens）
   ┌────────────────────────────┐
   │████░░░░████░░░░░░░░████░░░░│  ← 一行代表 Q 的一个 block
   │░░░░████░░░░████░░░░░░░░████│
Q  │████░░░░████░░░░████░░░░░░░░│     ████ = 计算（dense block）
序  │░░░░████░░░░████░░░░████░░░░│     ░░░░ = 跳过（sparse block）
列  │████░░░░████░░░░░░░░████░░░░│
   │░░░░████░░░░████░░░░░░░░████│
   └────────────────────────────┘

sparse_map: [B, H, Q_block_num, K_block_num]  (bool)
  Q_block_num = ceil(L / BLKQ)
  K_block_num = ceil(L / BLKK)
```

**为什么是块级而不是元素级？**

GPU 的 SRAM tile 大小决定了最小计算粒度。Flash Attention 内部就是以 128×128 的 tile 为单位做计算的。块稀疏直接跳过整个 tile，与 GPU 硬件完美匹配。

## 9. 单体稀疏实现：SLA Attention

**核心文件**：`lightx2v/common/ops/attn/sla_attn.py`

SLA（Sparse Linear Attention）是框架中最灵活的单体稀疏实现。它的特点是：**Mask 生成策略固定**（基于 Q·K 块均值的 TopK），但**计算后端可选**（triton/sage2/sage3/fa4/magi）。

### 9.1 Mask 生成：TopK 块选择

```python
# 核心思路（sla_util.py 中的 get_block_map）：
# 1. 把 Q, K 按块切分
# 2. 计算每个 Q 块均值与每个 K 块均值的点积 → [H, Q_blocks, K_blocks]
# 3. 每行取 TopK 个最大的块作为需要计算的块
sparse_map, lut, real_topk = get_block_map(
    q, k,                    # [B, H, L, D]
    topk_ratio=self.topk,    # 如 0.2（保留 20% 的块）
    BLKQ=128, BLKK=128,
)
# sparse_map: [B, H, Q_block_num, K_block_num]  布尔 mask
# lut: [B, H, Q_block_num, topk]  每行保留的 K_block 索引
```

### 9.2 计算后端路由

```python
class SlaAttnWeight(AttnWeightTemplate):
    sparsity_ratio = 0.8       # 类属性，由配置注入
    operator = "triton"        # 类属性，决定计算后端

    def __init__(self):
        self.topk = 1 - self.sparsity_ratio

        # 根据 operator 选择 apply 函数
        if self.operator == "triton":
            self.apply_func = self.apply_triton
        elif self.operator == "sage2":
            self.apply_func = self.apply_sage2
        elif self.operator == "sage3":
            self.apply_func = self.apply_sage3
        elif self.operator == "fa4":
            self.apply_func = self.apply_fa4
        elif self.operator == "magi":
            self.apply_func = self.apply_magi

    def apply(self, q, k, v, ...):
        return self.apply_func(q, k, v, ...)
```

**各后端的 sparse mask 消费方式不同**：

| 后端 | mask 消费格式 | 说明 |
|------|-------------|------|
| `triton` | `sparse_map` + `lut` | 自研 Triton kernel，直接用 LUT 索引 |
| `sage2` | `incremental_lut` + `valid_block_num` | SageAttention2 的 block sparse API |
| `sage3` | `ordinal_lut` + `valid_block_num` | SageAttention3 的 block sparse API |
| `fa4` | `full_block_idx` + `full_block_cnt` | FlashAttention4 的 block sparse API |
| `magi` | `q_ranges` + `k_ranges` | Magi 的 flex flash attention API |

**配置示例**：

```json

    "self_attn_1_type": "sla_attn",
    "sla_attn_setting": {
        "sparsity_ratio": 0.8,

    }
}

// SLA + SageAttention3 后端
{
    "self_attn_1_type": "sla_attn",
```

## 10. 单体稀疏实现：Sparge Attention

**核心文件**：`lightx2v/common/ops/attn/sparge_attn.py`

Sparge Attention 使用 `spas_sage_attn` 库的 `meansim_topk` 方法——Mask 生成与计算在一个 CUDA kernel 中完成。

```python
@ATTN_WEIGHT_REGISTER("sparge_attn")
class SpargeAttnWeight(AttnWeightTemplate):
    sparsity_ratio = 0.8

    def apply(self, q, k, v, ...):
        q, k, v = q.unsqueeze(0), k.unsqueeze(0), v.unsqueeze(0)
        q, k, v = q.transpose(1, 2), k.transpose(1, 2), v.transpose(1, 2)
        # Layout: [B, H, L, D]
        attn_out = spas_sage_attn.core.spas_sage2_attn_meansim_topk_cuda(
            q, k, v, topk=self.topk
        )
        return attn_out.permute(2, 1, 3, 0).contiguous().view(N, H * D)
```

**与 SLA 的核心区别**：SLA 用 Q·K 块均值的**点积**选 TopK；Sparge 用 Q·K 的**余弦相似度（mean-sim）**选 TopK。余弦相似度更稳定（不受向量模长影响），但需要额外的归一化计算。

## 11. 单体稀疏实现：spas_sage_attn2 / spas_sage_attn3 / spas_flash_attn4

这三个是"**Mask 策略 + 计算后端**打包在一起"的实现：

```python
# sage_attn.py:101
@ATTN_WEIGHT_REGISTER("spas_sage_attn2")
class SparseSageAttn2Weight(AttnWeightTemplate):
    sparsity_ratio = 0.8
    sparse_mode = "sla_mode"     # mask 策略: "sla_mode" 或 "sparge_mode"

    def apply(self, q, k, v, ...):
        if self.sparse_mode == "sla_mode":
            sparse_map, lut, _ = get_block_map(q, k, topk_ratio=self.topk, ...)
        elif self.sparse_mode == "sparge_mode":
            smooth_k = k - k.mean(dim=-2, keepdim=True)
            sparse_map = get_block_map_meansim(q, smooth_k, topk=self.topk, ...)
        # 用 SageAttention2 的 block sparse kernel 计算
        lut, valid_block_num = block_map_incremental_lut_triton(sparse_map)
        x = sage2_block_sparse_attn(q, k, v, lut, valid_block_num, ...)
        return x
```

**三者的差异仅在计算后端**：

| 注册名 | Mask 策略 | 计算后端 |
|--------|----------|---------|
| `spas_sage_attn2` | SLA 或 Sparge | SageAttention2 block sparse |
| `spas_sage_attn3` | SLA 或 Sparge | SageAttention3 block sparse |
| `spas_flash_attn4` | SLA 或 Sparge | FlashAttention4 block sparse |

## 12. 单体稀疏实现：结构化 Mask（Nbhd / SVG / Draft）

前面的稀疏实现（SLA/Sparge）是**数据驱动**的——Mask 依赖当前的 Q·K 值，每个 step 的 Mask 都不同。以下三种实现使用**结构化 Mask**——Mask 由视频的时空结构预先确定。

### 12.1 Nbhd Attention（邻域注意力）

**核心文件**：`lightx2v/common/ops/attn/nbhd_attn.py`

核心思想：视频中相邻帧的 token 之间的注意力更重要。Mask 是一个**对角带状**结构，带宽随帧距递减。

```python
# nbhd_attn.py:19
def generate_nbhd_mask(a, block_num, attnmap_frame_num,
                       coefficient=[1.0, 0.5, 0.056], min_width=1.0):
    """
    a: 每帧的 block 数
    coefficient: 各帧距的带宽系数（第0帧距=1.0，第1帧距=0.5，第2帧距=0.056...）
    """
    # 1. Attention sink: 第一帧的所有 token 始终被所有 Q 关注
    mask_sink = j_indices <= a

    # 2. 对角带: 帧距越远，带宽越窄
    for interval in range(0, attnmap_frame_num):
        width = max(min_width, coefficient[interval] * a)
        # 以帧距 interval 的对角线为中心，宽度为 width 的带
        mask_sparse |= (在对角线 ± width 范围内)

    return mask_sink | mask_sparse
```

**优点**：Mask 只需计算一次（序列长度不变时缓存），无需每步重新计算 Q·K 相关性。

### 12.2 SVG Attention

**核心文件**：`lightx2v/common/ops/attn/svg_attn.py`

SVG（Sparse Video Generation）使用**per-head 自适应** Mask：每个 head 从"空间优先"和"时间优先"两种 Mask 中选择 MSE 更小的那个。

```
空间优先 Mask: 同一帧内的 token 互相关注，跨帧稀疏
时间优先 Mask: 同一空间位置的不同帧互相关注，跨空间稀疏
```

每个 step 采样少量行计算 MSE 来选择最优 Mask（类似 A/B 测试），开销很低。

### 12.3 Draft Attention

**核心文件**：`lightx2v/common/ops/attn/draft_attn.py`

Draft Attention 的思路最直观：先用 **2D 平均池化**降低空间分辨率，在低分辨率上计算完整 Attention Map，然后通过 percentile 阈值生成稀疏 Mask，最后在原始分辨率上执行稀疏计算（使用 Magi kernel）。

```python
def apply(self, q, k, v, ..., block_idx=0, scheduler=None):
    if block_idx < 1:
        # 第一层用全注意力（Attention sink 层）
        return flash_attn_varlen_func(q, k, v, ...)

    # 1. 2D 平均池化采样
    attn = self.sample_qk_attention_2d(q, k, frame_h, frame_w, pool_h, pool_w)
    # 2. Per-head percentile 阈值
    mask = self.attention_percentile_mask_headwise(attn, 1 - self.sparsity_ratio)
    # 3. 添加 sink mask（第一帧始终关注）+ 对角 mask（同帧始终关注）
    mask[:, :, :mask_size_per_frame] = True
    mask |= (block_indices[:, None] == block_indices[None, :])
    # 4. 用 Magi kernel 执行稀疏注意力
    out = magi_ffa_func(q_reorged, k_reorged, v_reorged, q_ranges, k_ranges, ...)
```

## 14. GeneralSparseAttnWeight：组装器

**核心文件**：`lightx2v/common/ops/attn/general_sparse_attn.py`

```python
@ATTN_WEIGHT_REGISTER("general_sparse_attn")
class GeneralSparseAttnWeight(AttnWeightTemplate):
    sparse_mask_generator = None    # 类属性，由配置注入
    sparse_operator = None          # 类属性
    sparse_setting = {}
    operator_setting = {}

    def __init__(self):
        self.config = {}
        self._setup_operator()           # 从注册器取出 Operator 实例
        self._setup_mask_generator()     # 从注册器取出 MaskGenerator 实例

    def _setup_operator(self):
        self.operator = SPARSE_OPERATOR_REGISTER[self.sparse_operator](
            self.operator_setting
        )

    def _setup_mask_generator(self):
        self.mask_generator = SPARSE_MASK_GENERATOR_REGISTER[self.sparse_mask_generator](
            self.operator.q_block_size,  # 从 operator 获取块大小（确保匹配）
            self.operator.k_block_size,
            self.sparse_setting,
            self.attnmap_frame_num,
        )

    def apply(self, q, k, v, ...):
        # 1. 生成 sparse mask
        mask = self.mask_generator(q, k)
        # 返回: [B, H, Q_block_num, K_block_num]

        # 2. 可选的 reorg（如 SVG 的 per-head placement）
        q, k, v = self.mask_generator.reorg(q, k, v)

        # 3. 用 operator 执行稀疏计算
        out = self.operator(q, k, v, mask, ...)

        # 4. 可选的 restore（逆 reorg）
        out = self.mask_generator.restore(out)
        return out
```

## 15. Mask Generator 详解

**核心文件**：`lightx2v/common/ops/attn/sparse_mask_generator.py`

所有 Mask Generator 继承自 `GeneralMaskGenerator`：

```python
class GeneralMaskGenerator(ABC):
    def __init__(self, q_block_size=128, k_block_size=128,
                 sparse_setting={}, attnmap_frame_num=None):
        ...

    @abstractmethod
    def __call__(self, q, k):
        """返回 sparse_map: [B, H, Q_block_num, K_block_num]"""
        pass

    def reorg(self, q, k, v):
        """可选：重排输入序列（如 SVG 的 per-head placement）"""
        return q, k, v

    def restore(self, out):
        """可选：恢复原始顺序"""
        return out
```

**四种 Mask Generator 对比**：

| 名称 | 注册 key | Mask 特性 | 是否数据驱动 | 需要 reorg/restore |
|------|---------|-----------|------------|-------------------|
| SLA | `sla_mask_generator` | Q·K 块均值 TopK | 是（每步变化） | 否 |
| Sparge | `sparge_mask_generator` | Q·K 余弦相似度 TopK | 是（每步变化） | 否 |
| Nbhd | `nbhd_mask_generator` | 帧距衰减的对角带 | 否（可缓存） | 否 |
| SVG | `svg_mask_generator` | Per-head 空间/时间自适应 | 部分（head 选择） | 是（per-head 重排） |

**Nbhd Mask Generator 的缓存机制**：

```python
class NbhdMaskGenerator(GeneralMaskGenerator):
    seqlen = None      # 类变量
    mask = None        # 类变量

    def __call__(self, q, k):
        seqlen, _, _ = q.shape
        if seqlen == NbhdMaskGenerator.seqlen:
            return NbhdMaskGenerator.mask    # 命中缓存，直接返回
        # 首次或序列长度变化时才计算
        mask = generate_nbhd_mask(...)
        NbhdMaskGenerator.seqlen = seqlen
        NbhdMaskGenerator.mask = mask
        return mask
```

结构化 Mask 只依赖序列长度和视频尺寸，不依赖 Q·K 的实际值，因此可以跨 step 缓存。

## 16. Sparse Operator 详解

**核心文件**：`lightx2v/common/ops/attn/sparse_operator.py`

所有 Operator 的接口统一为：

```python
def __call__(self, q, k, v, mask,
             cu_seqlens_q=None, cu_seqlens_kv=None,
             max_seqlen_q=None, max_seqlen_kv=None, **kwargs):
    """
    q, k, v: [L, H, D]
    mask: [B, H, Q_block_num, K_block_num] (bool)
    返回: [L, H*D]
    """
```

**七种 Operator 对比**：

| 注册 key | 类名 | 底层 kernel | 块大小 | 特点 |
|---------|------|-----------|-------|------|
| `sla_triton_operator` | SlaTritonOperator | 自研 Triton kernel | 128×128 | 最通用，无额外依赖 |
| `spas_sage2_operator` | SparseSageAttentionV2Operator | SageAttn2 sparse | 64×128 或 128×64 | INT8 Q·K |
| `spas_sage3_operator` | SparseSageAttentionV3Operator | SageAttn3 sparse | 128×128 | Blackwell 专用 |
| `spas_fa4_operator` | SparseFlashAttentionV4Operator | FlashAttn4 sparse | 128×128 | FA4 原生 block sparse |
| `magi_operator` | MagiOperator | Magi flex_flash_attn | 128×128 | 支持任意 Q/K 范围 |
| `flex_block_operator` | FlexBlockOperator | flex_block_attn | 128×128 | FlexAttention 变体 |
| `flashinfer_operator` | FlashinferOperator | FlashInfer | 128×128 | 缓存 plan，支持变长 |

**mask 消费方式差异**：各底层 kernel 对 sparse mask 的表示格式不同，Operator 负责做格式转换：

```python
# SlaTritonOperator: mask → topk LUT
mask = mask.int()
topk = int(mask.sum(dim=-1).max().item())
lut = torch.topk(mask, topk, dim=-1, sorted=False).indices
out = _attention.apply(q, k, v, mask, lut, topk, BLKQ, BLKK)

# SparseSageAttentionV2Operator: mask → incremental LUT
lut, valid_block_num = block_map_incremental_lut_triton(mask)
out = sage2_block_sparse_attn(q, k, v, lut, valid_block_num, ...)

# SparseFlashAttentionV4Operator: mask → block index/count
full_block_idx, full_block_cnt = block_map_ordinal_lut_triton(mask)
out, _ = flash_attn_func_v4(q, k, v, full_block_cnt=full_block_cnt, ...)

# MagiOperator: mask → q_ranges/k_ranges
q_ranges, k_ranges = self.generate_qk_ranges(mask, q_block_size, ...)
out = magi_ffa_func(q, k, v, q_ranges=q_ranges, k_ranges=k_ranges, ...)
```

## 18. Ulysses Attention

**核心文件**：`lightx2v/common/ops/attn/ulysses_attn.py`

当单卡 GPU 显存不够时，可以用序列并行（Sequence Parallel）：把视频序列切分到多张 GPU 上，每张 GPU 只处理一部分 token。但 Attention 需要全局 Q·K 交互，不能简单切分——这就是 Ulysses Attention 解决的问题。

**核心思想**：把"按序列维度切分"转换为"按 head 维度切分"。

```
切分前（单卡）：
  Q: [L, H, D]  每张卡有 L/N 个 token, H 个 heads

All-to-All 通信后：
  Q: [L, H/N, D]  每张卡有 L 个 token, H/N 个 heads
                   ↳ 现在可以对完整序列做 Attention（只处理部分 heads）

Attention 计算后 All-to-All 回来：
  O: [L/N, H, D]  恢复为按序列切分
```

```python
@ATTN_WEIGHT_REGISTER("ulysses")
class UlyssesAttnWeight(AttnWeightTemplate):
    def apply(self, q, k, v, slice_qkv_len, cu_seqlens_qkv,
              attention_module=None, attention_type="flash_attn2",
              seq_p_group=None, use_fp8_comm=False, ...):
        world_size = dist.get_world_size(seq_p_group)
        shard_heads = heads // world_size

        # 1. 分离图像和文本部分
        img_q = q[:img_qkv_len, :, :]     # 图像 token（按序列切分）
        txt_q = q[img_qkv_len:, :, :]     # 文本 token（不切分，每卡都有完整副本）

        # 2. 图像部分重排: [shard_seq, H, D] → [shard_seq, world_size, shard_heads, D]
        img_q = img_q.reshape(img_qkv_len, world_size, shard_heads, hidden_dims)

        # 3. All-to-All: 按序列切分 → 按 head 切分
        #    每张卡：[shard_seq, world_size, shard_heads, D]
        #         → [global_seq, shard_heads, D]（拼接其他卡的同 head 数据）
        img_q = all2all_head2seq(img_q, seq_p_group)
        #    ↳ 底层用 dist.all_to_all

        # 4. 拼接文本 token（文本不参与切分，直接拼在图像后面）
        q_full = torch.cat([img_q, txt_q], dim=0)

        # 5. 用底层注意力算子（如 flash_attn2）计算
        attn_out = attention_module.apply(q_full, k_full, v_full, ...)

        # 6. All-to-All 回来: 按 head 切分 → 按序列切分
        img_out = all2all_head2seq(img_out, seq_p_group, reverse=True)
        return torch.cat([img_out, txt_out], dim=0)
```

**Ulysses 在调用链中的位置**：

```
WanSelfAttention (权重层)
  ├── self_attn_q/k/v/o: MMWeight      (Q/K/V/O 投影)
  ├── self_attn_1: FlashAttn2Weight     (底层注意力算子)
  └── self_attn_1_parallel: UlyssesAttnWeight  (序列并行包装器)
                                 │
                     调用 self_attn_1.apply()
                     作为底层计算后端
```

推理时，序列并行模式下调用 `self_attn_1_parallel.apply()`，非并行模式下直接调用 `self_attn_1.apply()`（见 `wan/infer/transformer_infer.py:219-245`）。

**通信优化**：Ulysses 支持 FP8/FP4 量化通信（`use_fp8_comm`/`use_fp4_comm`），在 All-to-All 传输前量化、传输后反量化，减少 50-75% 的通信量。
