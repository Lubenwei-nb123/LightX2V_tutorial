# LightX2V 推理框架深度剖析课程大纲

## 课程目标
通过 15 节课深入理解 LightX2V 框架设计，最终能够：
- 掌握框架核心架构与设计模式
- 理解各优化技术的实现原理
- 具备快速适配新模型的能力
- 能够集成和开发新的优化手段
- 深入理解服务化与分离部署的完整链路

---

## 第一部分：架构设计（第1-5课）

### 第1课：整体架构与设计哲学
- 框架分层设计：Pipeline → Runner → Components
- 核心抽象：为什么这样设计？
- 注册机制：RUNNER_REGISTER 的实现与作用
- 配置驱动：JSON 配置如何映射到代码执行
- 扩展点设计：哪些地方预留了扩展能力

### 第2课：Pipeline 层设计
- LightX2VPipeline 的职责边界
- model_cls 到 Runner 的映射机制
- Offload 配置的传递与生效
- Generator 的创建与参数注入
- 多任务支持（T2V/I2V/T2I）的统一抽象

### 第3课：Runner 层设计
- BaseRunner → DefaultRunner → 具体Runner 的继承体系
- Runner 的核心职责：组件编排与生命周期管理
- 组件初始化顺序与依赖关系
- 推理循环的设计（encode → denoise loop → decode）
- WanRunner vs HunyuanVideoRunner 的差异分析

### 第4课：网络层设计（DiT）
- base_model.py 的抽象设计
- 三阶段推理：pre_infer / transformer_infer / post_infer
- 权重加载机制：weights/ 目录的组织方式
- infer/ 与 weights/ 的分离设计意图
- 如何支持不同的 DiT 变体

### 第5课：编码器与调度器设计
- 输入编码器的抽象：文本编码器、图像编码器
- 编码器的懒加载与 Offload 支持
- 调度器接口设计：scheduler.py
- Flow Matching vs DDPM 的调度器实现差异
- VAE 编解码的集成方式

---

## 第二部分：优化技术实现（第6-9课）

### 第6课：Offload 机制实现
- 显存压力的来源分析
- Phase 级 Offload：整个组件的迁移
- Block 级 Offload：DiT Block 粒度的迁移
- Disk Offload：磁盘缓存策略
- Offload 调度器的实现细节
- 如何为新模型添加 Offload 支持

### 第7课：注意力算子体系
- 注意力算子的统一接口设计（AttnWeightTemplate）
- Dense 注意力：Flash Attention 2/3/4、Sage Attention 2/3、Torch SDPA
- Sparse 注意力单体实现：SLA、Sparge、Nbhd、SVG、Draft
- GeneralSparseAttn 框架：Mask Generator × Operator 双注册器解耦
- 序列并行注意力：Ulysses 的 All-to-All 通信设计
- 如何集成新的注意力算子

### 第8课：并行策略实现
- 并行配置入口：`set_parallel_config` 与 `device_mesh` 的构建
- CFG 并行：正负样本的双路前向与 All-Gather 合并
- 序列并行（Ulysses）：序列维度 ↔ Head 维度的 All-to-All 通信
  - 图文分离处理：为什么文本 token 不切分
  - FP8/FP4 量化通信：通信量压缩 50-75%
  - Tensor Fusion：QKV 合并通信优化
- CFG × Seq 二维并行：`device_mesh` 的 2D 网格组合
- Tensor 并行：权重切分与通信模式
- 各并行策略的适用场景与显存/通信开销分析
- 并行配置示例与实战调优

### 第9课：量化技术实现
- 量化的基本原理回顾
- FP8 量化的实现方式
- NVFP4 量化支持
- TorchAO / GGUF 格式支持
- 量化与 Offload 的交互：FP8 权重在 CPU↔GPU 传输中的优势
- 量化与其他优化的组合

---

## 第三部分：服务化与分离部署（第10-13课）

### 第10课：服务化架构总览
- Server 模块的整体架构：API → Service → Worker → Pipeline
- FastAPI 应用的启动流程（`__main__.py` → `main.py` → `server.py`）
- 请求的完整生命周期：从 HTTP 到视频生成
- 配置体系：`ServerConfig` 与 JSON 配置的映射
- Schema 设计：请求/响应的数据模型（`schema.py`）
- 与 CLI 推理的差异：为什么服务化需要额外的抽象层

### 第11课：任务管理与推理调度
- TaskManager 的设计：任务队列、状态机、并发控制
- InferenceService 与 InferenceWorker 的分工
- 单线程推理 + 异步 API 的设计选择（为什么不用多线程推理）
- 请求排队与拒绝策略（`max_queue_size`）
- Generation Service 层：VideoGenerationService / ImageGenerationService
- 文件服务：生成结果的存储与访问（`FileService`）
- 监控与 Metrics：Prometheus 指标的采集与暴露

### 第12课：分离部署架构设计（disagg 模块）
- 为什么需要分离部署：Encoder/Transformer/Decoder 的资源特性差异
- disagg 模块的整体架构：Controller → Encoder/Transformer/Decoder 服务
- Protocol 设计：服务间通信的数据协议（`protocol.py`）
- 服务基类设计：`BaseDisaggService` 的抽象
- Controller 的调度逻辑：请求分发与结果收集
- Encoder / Transformer / Decoder 各阶段服务的实现
- Round-Robin 调度器：多实例负载均衡（`scheduler/`）
- 配置示例：以 Wan T2V 为例的四服务配置

### 第13课：分离部署通信层（RDMA 与 Mooncake）
- 传输层的挑战：视频推理的 latent 张量有多大
- RDMA 通信的基础原理：零拷贝、内核旁路
- `rdma_buffer.py`：RDMA 缓冲区的注册与管理
- `rdma_client.py` / `rdma_server.py`：RDMA 传输的客户端与服务端
- `conn.py`：连接管理与重连机制
- Mooncake 集成：`mooncake.py` 的适配层
- 传输层与服务层的解耦：如何切换 TCP / RDMA / Mooncake
- Monitor 与健康检查：`monitor.py` 的实现
- 端到端部署实战：从启动到请求的完整流程
